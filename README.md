# بنیان

معماری لایه‌ای افزونه‌محور برای ساخت وب‌اپلیکیشن با Vanilla JS، HTML، و CSS.

[![هوش‌مصنوعی](https://img.shields.io/badge/Built%20with-AI-blueviolet)](#)
[![نسخه](https://img.shields.io/badge/version-0.6-blue.svg)](package.json)
[![مجوز](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

> این معماری با کمک مدل هوش مصنوعی Claude Sonnet 4.6 و بخش کنش‌ور وبگاه [arena](https://arena.ai) نوشته شده است.

## چیست؟

بنیان یک معماری است، نه یک کتابخانه. هیچ چیزی برای نصب وجود ندارد. بنیان مجموعه‌ای از قراردادها، مرزها، و ساختارهایی است که به شما کمک می‌کند یک اپلیکیشن قابل گسترش، قابل نگهداری، و قابل حمل بسازید.

## چرا بنیان؟

- **بدون فریم‌ورک.** فقط Vanilla JS، HTML، و CSS استاندارد.
- **جداسازی کامل.** منطق کسب‌وکار هیچ‌چیزی از رابط کاربری نمی‌داند.
- **افزونه‌محور.** هر ویژگی یک افزونه مستقل است با مانیفست و نقطه ورود مشخص.
- **قابل حمل.** هسته و افزونه‌ها را می‌توان در پروژه‌های مختلف استفاده کرد.
- **چندپلتفرمی.** بدون تغییر در کد روی مرورگر، موبایل (Capacitor)، و دسکتاپ (Tauri) اجرا می‌شود.
- **آداپتورمحور.** ذخیره‌سازی، احراز هویت، و سایر وابستگی‌های خارجی قابل تعویض هستند.
- **رابط کاربری مستقل.** UI مانیفست دارد، سرویس‌ها را مستقیم صدا می‌زند، و از طریق رویداد با افزونه‌ها ارتباط برقرار می‌کند.

## اصول بنیادین

- سادگی بر پیچیدگی مقدم است
- مرزها دقیق و مستند هستند
- منطق کسب‌وکار باید مستقل باشد
- هسته باید کوچک و ناآگاه باقی بماند
- قابلیت حمل پیش‌فرض است
- توسعه‌دهنده مسئول است
