# مانیفست (Manifest)

## مقدمه

مانیفست تنها منبع معتبر برای شناخت یک افزونه است. هر افزونه باید یک مانیفست داشته باشد. هسته، افزونه‌های دیگر، و هر کسی که با افزونه کار می‌کند، فقط از طریق مانیفست می‌فهمد این افزونه چیست، چه سرویس‌هایی دارد، چه رویدادهایی منتشر می‌کند، و به چه چیزهایی نیاز دارد. هر چیزی که در مانیفست اعلام نشده باشد از دیدگاه سیستم وجود ندارد.

## ساختار مانیفست

هر مانیفست از شش بخش تشکیل شده است.

### هویت

اطلاعات پایه افزونه را مشخص می‌کند:

**name:** نام منحصر به فرد افزونه با فرمت `vendor.name`. هیچ دو افزونه‌ای در یک سیستم نمی‌توانند نام یکسان داشته باشند.

**version:** نسخه افزونه. جزئیات کامل در سند `versioning.fa.md` آمده است.

**architecture:** نسخه معماری بنیان که این افزونه برای آن نوشته شده. هسته از این مقدار برای بررسی سازگاری استفاده می‌کند.

**description:** توضیح کوتاهی درباره کار افزونه. این توضیح برای انسان است، نه برای هسته.

### نوع

نوع افزونه را مشخص می‌کند. پنج مقدار ممکن است:

**infrastructure:** سرویس‌های پایه‌ای فراهم می‌کند که بقیه افزونه‌ها به آنها وابسته هستند. نمونه‌ها: ذخیره‌سازی، مسیریابی، مدیریت حالت، لاگ، چندزبانگی.

**product:** منطق اصلی اپلیکیشن را پیاده‌سازی می‌کند. بدون این افزونه‌ها اپلیکیشن هدف اصلی‌اش را انجام نمی‌دهد.

**feature:** قابلیت اضافی فراهم می‌کند که اپلیکیشن بدون آن هم کار می‌کند. نمونه‌ها: جستجو، اشتراک‌گذاری، نمودار.

**platform:** سازگاری با محیط اجرا را فراهم می‌کند. نمونه‌ها: IndexedDB، SQLite، FileSystem.

**transport:** انتقال پیام بین سیستم و دنیای بیرون را مدیریت می‌کند. نمونه‌ها: HTTP، WebSocket.

### وابستگی‌ها

سرویس‌هایی که افزونه به آنها نیاز دارد اینجا اعلام می‌شوند. وابستگی‌ها دو نوع هستند:

**وابستگی اجباری:** افزونه بدون این سرویس نمی‌تواند کار کند. اگر سرویس در زمان راه‌اندازی موجود نباشد، افزونه غیرفعال می‌شود.

**وابستگی اختیاری:** اگر سرویس موجود باشد افزونه از آن استفاده می‌کند. اگر موجود نباشد، افزونه با قابلیت کاهش‌یافته کار می‌کند.

هر افزونه فقط می‌تواند از سرویس‌هایی استفاده کند که اینجا اعلام کرده. درخواست به سرویس اعلام‌نشده توسط هسته رد می‌شود.

### سرویس‌های ارائه‌شده

اگر افزونه سرویسی برای استفاده دیگران فراهم می‌کند، باید اینجا اعلام شود. برای هر سرویس باید مشخص شود:

**name:** نام سرویس که در رجیستری ثبت می‌شود.

**description:** توضیح کوتاهی درباره کار سرویس.

اگر افزونه هیچ سرویسی ارائه نمی‌دهد این بخش خالی می‌ماند.

### رویدادهای منتشرشده

رویدادهایی که افزونه از طریق گذرگاه رویداد منتشر می‌کند اینجا اعلام می‌شوند. برای هر رویداد باید مشخص شود:

**name:** نام رویداد با فرمت `pluginname:eventname`.

**description:** توضیح کوتاهی درباره اینکه این رویداد چه زمانی منتشر می‌شود.

این اعلام به افزونه‌های دیگر کمک می‌کند بدون نگاه کردن به کد بفهمند به چه رویدادهایی می‌توانند گوش بدهند.

اگر افزونه هیچ رویدادی منتشر نمی‌کند این بخش خالی می‌ماند.

### پیکربندی پیش‌فرض

اگر افزونه پیکربندی‌پذیر است، مقادیر پیش‌فرض اینجا تعریف می‌شوند. توسعه‌دهنده می‌تواند این مقادیر را در فایل پیکربندی پروژه بازنویسی کند. مقادیر پروژه همیشه اولویت دارند.

اگر افزونه پیکربندی ندارد این بخش خالی می‌ماند.

## نمونه مانیفست

یک نمونه واقعی از مانیفست افزونه تراکنش‌ها در یک نرم‌افزار مالی:

```json
{
  "name": "myfinance.transactions",
  "version": "1.2.0",
  "architecture": "1.0",
  "description": "مدیریت تراکنش‌های مالی",
  "type": "product",
  "dependencies": {
    "required": [
      "myfinance.storage.service",
      "myfinance.accounts.service"
    ],
    "optional": [
      "myfinance.notifications.service"
    ]
  },
  "provides": [
    {
      "name": "myfinance.transactions.service",
      "description": "دسترسی به داده‌ها و منطق تراکنش‌ها"
    }
  ],
  "events": [
    {
      "name": "transactions:saved",
      "description": "وقتی یک تراکنش جدید با موفقیت ذخیره می‌شود"
    },
    {
      "name": "transactions:deleted",
      "description": "وقتی یک تراکنش حذف می‌شود"
    }
  ],
  "config": {
    "maxTransactionsPerPage": 50,
    "defaultCurrency": "IRR"
  }
}
```

## قوانین مانیفست

**مانیفست تنها منبع معتبر است.** هسته افزونه‌ای را که مانیفست ناقص یا نامعتبر دارد راه‌اندازی نمی‌کند.

**وابستگی‌های پنهان ممنوع است.** اگر افزونه به سرویسی نیاز دارد که در مانیفست اعلام نشده، این یک خطای معماری است. هسته چنین دسترسی را رد می‌کند.

**مانیفست تغییر نمی‌کند.** مانیفست در زمان راه‌اندازی خوانده می‌شود و تا زمان خاموش‌شدن افزونه ثابت می‌ماند. افزونه نمی‌تواند در زمان اجرا وابستگی یا دسترسی جدید اضافه کند.

**نسخه‌بندی اجباری است.** هر تغییر در افزونه باید با به‌روزرسانی نسخه همراه باشد. جزئیات در سند `versioning.fa.md` آمده است.
