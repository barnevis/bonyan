# نام‌گذاری (Naming)

## مقدمه

قراردادهای نام‌گذاری در بنیان ساده و کم هستند. هدف این است که هر کسی که کد را می‌خواند بلافاصله بفهمد با چه چیزی طرف است و از کجا آمده.

## نام‌گذاری افزونه

همه افزونه‌ها از یک فرمت دو بخشی پیروی می‌کنند:

```
vendor.name
```

**vendor** نام شخص، تیم، یا سازمانی است که افزونه را نوشته. این بخش از تداخل نام بین افزونه‌های مختلف جلوگیری می‌کند.

**name** نام کوتاه و توصیفی افزونه است. باید بیانگر کار افزونه باشد.

نمونه‌ها:
- `myfinance.transactions`
- `myfinance.accounts`
- `myfinance.storage`
- `acme.reports`

هیچ دو افزونه‌ای در یک سیستم نمی‌توانند نام یکسان داشته باشند. هسته در زمان راه‌اندازی این یکتایی را بررسی می‌کند.

## نام‌گذاری سرویس

سرویس‌ها همان نام افزونه‌ای را دارند که آنها را فراهم می‌کند به علاوه پسوند `.service`:

```
vendor.name.service
```

نمونه‌ها:
- `myfinance.transactions.service`
- `myfinance.storage.service`
- `acme.reports.service`

## نام‌گذاری رویداد

رویدادها از فرمت زیر پیروی می‌کنند:

```
pluginname:eventname
```

**pluginname** نام بخش دوم نام افزونه است، بدون vendor.

**eventname** نام رویداد است و باید بیانگر چیزی باشد که اتفاق افتاده، نه چیزی که باید انجام شود.

نمونه‌ها:
- `transactions:saved`
- `transactions:deleted`
- `accounts:updated`
- `storage:ready`
- `core:plugin-failed`
- `core:plugin-crashed`

## قوانین کلی نام‌گذاری

**حروف کوچک و خط تیره.** همه نام‌ها با حروف کوچک نوشته می‌شوند. اگر نام چند کلمه دارد از خط تیره استفاده می‌شود. مثال: `myfinance.budget-reports`.

**نام‌های توصیفی.** نام باید بگوید افزونه یا رویداد چیست، نه چطور کار می‌کند.

**اختصار ممنوع است.** `transactions` درست است، `txn` اشتباه است. خوانایی مهم‌تر از کوتاهی است.

**نام‌های رویداد در زمان گذشته.** رویداد چیزی است که اتفاق افتاده. `saved` درست است، `save` اشتباه است. استثنا رویدادهای هسته هستند که با `core:` شروع می‌شوند.
